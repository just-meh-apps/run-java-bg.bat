# ☕ Java JAR Background Runner (Windows 10/11)

Windows 환경에서 자바 기반 도구(예: Bytecode Viewer, JD-GUI, APK Analyzer 등)를 실행할 때, 매번 함께 뜨는 **불필요한 검은색 CMD 콘솔 창을 제거하거나 최소화**하여 깔끔한 작업 환경을 제공하는 스크립트 모음입니다.

이 프로젝트는 특히 GUI 기반의 자바 애플리케이션을 자주 사용하는 개발자와 리버스 엔지니어를 위해 최적화되었습니다.

---

## ✨ 주요 특징

- **스마트 자바 경로 탐색 (Multi-Priority)**: 
  환경 변수가 꼬여있는 상황을 대비하여 1순위 사용자 정의 경로(`DEFAULT_JAVA_HOME`), 2순위 시스템 `JAVA_HOME`, 3순위 `JDK_HOME`을 순차적으로 자동 탐색합니다.
- **콘솔 창 제어 및 최소화**: 
  `javaw` 명령어를 사용하여 GUI 프로세스만 독립적으로 실행하며, `start /min` 옵션을 통해 실행 시 발생하는 CMD 창의 번쩍임을 작업 표시줄 수준으로 억제합니다.
- **완벽 무음 실행 지원**: 
  VBS 스크립트를 함께 제공하여, 단 0.1초의 창 노출도 없는 완벽한 백그라운드 실행이 가능합니다.
- **범용적인 경로 및 공백 처리**: 
  상대 경로와 절대 경로를 모두 지원하며, 윈도우 경로상의 공백이나 특수문자로 인한 오류를 방지하기 위해 따옴표 처리가 완벽히 되어 있습니다.
- **모듈형 엔진 구조**: 
  실행 엔진(`run_java_bg.bat`) 하나로 여러 개의 실행기(`run_sample.bat`)를 만들어 다양한 자바 도구(JAR)를 효율적으로 관리할 수 있습니다.

---

## 📂 파일 구성 및 역할

1. **`run_java_bg.bat` (핵심 엔진)**
   - 자바 실행 환경을 확인하고 실제 백그라운드 실행을 담당하는 메인 스크립트입니다.
   - 현재 디렉토리를 기억(`pushd`)한 후 실행 위치로 이동하여 작업을 수행하고 다시 복귀하는 안전한 로직을 포함합니다.
   - 자바 경로를 찾지 못하거나 인수가 누락된 경우 명확한 한국어 에러 메시지를 출력합니다.

2. **`run_sample.bat` (실행기)**
   - 사용자가 실행하고자 하는 특정 `.jar` 파일의 경로를 지정하는 파일입니다.
   - 이 파일을 복사해서 `run_bcv.bat`, `run_jdgui.bat` 등 여러 이름으로 확장해 사용할 수 있습니다.

3. **`run_sample.vbs` (완벽 무음 실행기)**
   - 윈도우 스크립트 엔진을 활용하여 CMD 창이 아예 뜨지 않도록 실행기를 호출합니다.
   - 시각적인 방해 없이 즉시 프로그램만 띄우고 싶을 때 사용합니다.

---

## 🚀 시작하기

### 1. 설치 (Setup)
- 본 저장소의 모든 파일(`.bat`, `.vbs`)을 다운로드하여 실행할 JAR 파일과 같은 폴더에 넣습니다. (파일들이 서로 다른 위치에 있어도 절대 경로를 지정하면 정상 작동합니다.)

### 2. 설정 (Configuration)
- `run_sample.bat` 파일을 메모장으로 열고 상단의 `TARGET` 변수값을 수정합니다.
  ```batch
  set "TARGET=.\your_application_name.jar"
  ```
- **(선택 사항)** 시스템 환경 변수를 사용하지 않으려면 `run_java_bg.bat`를 열어 `DEFAULT_JAVA_HOME`의 주석을 제거하고 직접 JDK 경로를 입력하세요.

### 3. 실행 (Execution)
- **가장 깔끔한 방법**: `run_sample.vbs`를 더블 클릭합니다.
- **일반적인 방법**: `run_sample.bat`를 더블 클릭합니다.

## 💻 시스템 요구 사항
